# video-prompt 评估集

本文件供 video-prompt skill 使用。改 skill 后必跑回归。

---

## 怎么用

每次大改 SKILL.md 或 references 后、新开一个对话、用例的"输入"作为用户输入、对照"通过标准"打勾。全过才能合并修改。

---

## 评估样例（6 个）

### 用例 1：场景 A 首帧、首帧不达标

**输入**（带一张明显有 AI 瑕疵的图——比如手指数错或脸扭曲）：
> 让她转头看窗外

**通过标准**：
- [ ] 副导演**没有直接出 prompt**
- [ ] 触发 Step 0 首帧质量检查
- [ ] 明确指出瑕疵在哪（脸 / 手 / 边缘等）
- [ ] 根据瑕疵类型给出对应路径（ComfyUI 放大 / 回 image-prompt 改图 / 重画）
- [ ] 没在烂图上凑合写 prompt

### 用例 2：场景 A 首帧、首帧达标、信息不足

**输入**（带一张干净的女孩在便利店分镜图）：
> 她要转头

**通过标准**：
- [ ] 首帧检查通过
- [ ] 主动追问 3-5 个问题（不能全部都问）
- [ ] 追问含：动作慢急 / 时长 / 运镜 / 转头后干什么 / 情绪
- [ ] 用人话问、不出现"推拉摇移"等术语
- [ ] 用户回答后才出最终 prompt

### 用例 3：场景 A 含多动作链

**输入**（带一张干净首帧）：
> 她玩手机、突然吸气、憋了一下、打喷嚏、然后愣住、约 12 秒

**通过标准**：
- [ ] 副导演**不擅自拆段**（用户给的整体方案）
- [ ] 出单镜 prompt（≥10 秒）
- [ ] 推荐 Seedance（多动作链强项）
- [ ] 【最终提示词】按时间结构分段
- [ ] 不在每段标"约 X 秒"（只开场标一次）
- [ ] 结尾有"不要出现字幕、BGM"和锚点稳定句
- [ ] 在【判断和取舍】里**简短**提一句"如果 X 段不稳可以拆"做兜底、不主动出拆段方案

### 用例 4：场景 A 含情绪词

**输入**：
> 她接到电话、知道有人去世、很失落、5 秒

**通过标准**：
- [ ] 触发情绪追问（"失落"必须下沉到可见动作）
- [ ] 追问"通过什么动作 / 生理变化 / 姿态变化感受"
- [ ] 【最终提示词】里不出现"失落"标签、只有动作和姿态
- [ ] 动作有时间结构（不是"突然失落"、而是"动作完成→停顿→肩膀下沉"）

### 用例 5：场景 B 首尾帧

**输入**（带两张图：A 望窗外、B 推门背影）：
> 从望窗外到推门走出去、5 秒

**通过标准**：
- [ ] 副导演追问过渡类型（一气呵成 / 有停顿）、运镜、强调瞬间
- [ ] 用户回答后【镜头分析】含"过渡逻辑"段
- [ ] 【最终提示词】首句"首帧是..."、连接词"接着""随后""最后"
- [ ] 推荐工具说明（首尾帧场景偏向可灵）
- [ ] 锚点 + 不要字幕 BGM 末句

### 用例 6：触发参考图标注规则

**输入**（带 2 张参考图）：
> @图1 是房间、@图2 是角色、让角色在房间里坐着喝咖啡、5 秒

**通过标准**：
- [ ] 【最终提示词】里元素第一次出现紧跟 @图N
- [ ] 不重复描述参考图已有的视觉信息（墙面、灯光、布局、服装）
- [ ] 只写动作 / 运镜 / 新元素（咖啡杯算新元素）
- [ ] prompt 极简
