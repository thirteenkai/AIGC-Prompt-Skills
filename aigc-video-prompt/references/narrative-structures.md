# 叙事结构与时间线（复杂场景才用）

本文档供 aigc-video-prompt skill 使用。默认最终提示词采用 `references/prompt-architecture.md` 的导演四块结构；本文件只处理需要更强叙事控制的两个技巧：**五段式叙事结构** 和 **分镜节拍 / 时间线分段标记**。

## ⚠️ 触发条件（必读，避免滥用）

四块结构默认使用，时间线和五段式不是默认。只有画面需要明确节拍、转场或叙事弧线时才加。**节拍不等于秒数**：时间把握不准时，用分镜节拍，不要硬写秒级。

| 触发条件 | 用哪个 |
| --- | --- |
| 用户明确要"讲个故事"（仿真人剧/微短剧/精品广告剧情段） | 五段式 |
| 视频时长 ≥ 10 秒 + 有人物或情绪弧线 + 用户希望精确控制每一拍 | 五段式 + 时间线 |
| 视频时长 ≥ 10 秒 + 多动作 / 多对白 + 时间分配没把握 | 分镜节拍，不写秒数 |
| 多角色对话戏 + 需要明确正反打调度 | 时间线（配合备选输出格式：结构化分镜） |
| 多角色对话戏 + 同一机位内错开台词和动作 | 分镜节拍，不写秒数 |
| 复杂动作戏 + 用户需要精准控制每个动作的时间窗口 | 时间线 |

**绝对不要套五段式**：

- ❌ 单一动作展示（一只猫伸懒腰、一杯咖啡冒热气、产品展示一周）
- ❌ 运镜测试 / 角色一致性测试
- ❌ UGC 直拍风（要的就是"没设计感"）
- ❌ 数字人口播（基本中近景 + 一段台词）
- ❌ 时长 ≤ 8 秒的单一镜头
- ❌ 用户没明确要叙事时，不要预防性套五段式

这些场景仍然使用导演四块结构，只是不写五段式和秒级时间线。

判断顺序：先问自己"这条视频有起承转合或强控制节拍吗"。没有就只用四块结构；有，再考虑时间线或五段式。

## 五段式叙事结构（Five-Part Spine）

把一段视频当一个**微型故事**拆，分成 5 个功能段落。本质是讲故事的"起承转合"套到时长上的节奏分配。

### 结构表（以 15 秒为例）

| 时间窗口 | 段落 | 干什么 | 镜头语言典型 |
| --- | --- | --- | --- |
| 0-3 秒 | **开场（Establishing）** | 广角定场，告诉观众"这是哪里" | 广角 / 远景 + 固定或缓推 |
| 3-7 秒 | **发展（Development）** | 主角出现，开始动 | 中景 + 跟拍 |
| 7-11 秒 | **高潮（Climax）** | 情绪 / 动作的最高点 | 切特写、节奏最快、光影最强 |
| 11-13 秒 | **转折（Transition）** | 情绪回落、收束 | 中景缓拉，过渡到结尾 |
| 13-15 秒 | **结尾（Resolution）** | 定格收尾，给观众"呼吸的出口" | 全景定格、淡出、定格特写 |

### 关键原则

1. **时长不是死的**：6-8 秒短视频按比例压缩（开场 1.5 秒、发展 2 秒、高潮 2 秒、转折 1 秒、结尾 1.5 秒）；10 秒视频则按 2-3-3-1-1 分。
2. **不要每段都写新主体**：主角名词保持一致，模型才不会把不同段当成不同人。例如全段都用"穿米色风衣的女孩"，不要改成"女子"或"她"再换回"女孩"。
3. **全局风格行放最前**：光线、色调、分辨率写在五段之前，不在每段重复。
4. **高潮段是光影描述的最高杠杆点**——这段把最好的光线/色调描述写满。
5. **结尾不引入新信息**：结尾是收束，新元素让观众没法消化。

### 适合的内容类型

✅ 仿真人剧片段（有人物 + 情绪起伏 + 时长 ≥ 10s）
✅ 精品广告剧情段（问题→展示→解决的弧线）
✅ MV 镜头段落
✅ 12 秒以上有完整叙事的内容

❌ 不要套到单一动作、口播、UGC、产品展示上

### 五段式 + 时间线标记的完整写法示例

```
【全局锚定】穿米色风衣的女孩、深夜便利店、王家卫情绪色调、电影感

[00:00-00:03] 广角定场镜头，雨后城市街道全景，冷蓝色调，便利店招牌在画面中央
[00:03-00:07] 中景跟拍，女孩从画面左侧入画，沿便利店外街道缓步前行，双手插在口袋
[00:07-00:11] 特写，女孩停下脚步抬头看向便利店玻璃门，眼神从疲惫转为温柔，暖光从侧面打在脸上
[00:11-00:13] 中景缓慢拉远，女孩推开玻璃门走入便利店，背影逐渐缩小
[00:13-00:15] 全景定格，便利店内冷白荧光灯，雨开始变大，画面渐暗

整段保持人物面部特征、服装、发型、便利店环境、光线氛围稳定一致。不要出现字幕、不要出现 BGM。
```

## 时间线分段标记（Time Markers）

把提示词从"静态描述"升级为"动态程序"。模型按秒级标记分配每段时间的画面内容。

### 四种格式（模型都能识别）

| 格式 | 写法 | 适合场景 |
| --- | --- | --- |
| **方括号时间码** | `[00:00-00:05] 镜头缓慢推进...` | 最正式，精确到秒，复杂叙事 |
| **秒级简写** | `0-5s: 镜头俯拍...` | 简洁，快速迭代 |
| **中文秒级** | `0～3 秒：特写、湿漉漉的猫爪踩在积水的石板路上` | 可读性最佳，**中文社区最常用** |
| **分镜节拍** | `分镜一：角色入画点单...` / `起手：...` | 时间分配没把握、只需要动作顺序 |
| **镜头编号** | `Shot 1: 广角建立镜头...` | 多镜头蒙太奇 |

推荐：**先选分镜节拍，再选秒级**。只有你能明确判断每段该占几秒，或用户要求卡点时，才用中文秒级。时间把握不准时，写分镜节拍比乱写 0-3 / 3-5 秒更稳。

### 时间标记的核心规则

1. **15 秒视频通常拆 3-5 个节拍**（不是越多越好）
2. **每个时间窗口只装一个运镜 + 一个主体动作**（违反法则 1 直接崩）
3. **主体名词必须一致**——前后用"the businesswoman / a professional woman"会被识别为两个人
4. **全局风格行放在最上方**，不在每段重复
5. **不要把不确定的时间写成确定数字**。不确定就用"分镜一 / 分镜二"或"起手 / 中段 / 收手"。

### 实测加分

合理使用时间标记能提升可控性。但前提是用对场景——用错反而拖累。错误秒数会让模型抢节奏、漏动作或台词错位。

## 提示词中音频与时间标记的结合（高阶）

Seedance 2.0 的双分支架构使音频和视频在同一生成过程中"长"出来——音频描述会反过来影响画面节奏。

把音频描述紧跟在时间标记之后写：

```
[00:07-00:11] 特写，角色停下脚步抬头看路灯。
环境音：雨声渐大，远处雷声滚动。
SFX：雨滴打在雨伞上的急促声响。
音乐：钢琴单音缓慢敲击，情绪从紧张转为释然。
```

四层音频结构（用户决定要不要用、不要预防性堆音频）：

| 层级 | 功能 | 例子 |
| --- | --- | --- |
| **Ambient（环境音）** | 建立空间感 | 雨打玻璃、城市远景、风吹窗帘 |
| **SFX（音效）** | 强化动作 | 脚步踩碎石、钥匙金属碰撞、门锁咔哒 |
| **Music（音乐）** | 驱动情绪 | 慢钢琴 + 大提琴底音，小调 |
| **Silence（静默）** | 制造张力 | 房间只剩时钟的滴答声 |

四层不要同时全开。一个 15 秒视频通常 ambient 全程在 + SFX 跟随特定动作 + music 在情绪段介入 + silence 用于高潮前停顿。

**默认策略**：默认不写 Music 层，避免模型自行生成背景音乐。SFX 和 Ambient 可以写进 prompt 让 Seedance 直接生成。

## 决策树：拿到需求后怎么选

```
用户的需求是？
│
├─ 单一动作 / 口播 / UGC / 产品展示
│   └─ 用导演四块结构，不用五段式、不用时间标记。
│
├─ 时长 < 10 秒、只是想"动起来"
│   └─ 用导演四块结构。只有动作多于 3 拍时，才在【镜头与动作】里用 2-3 段节拍。
│
├─ 时长 ≥ 10 秒 + 有情绪弧线 / 故事线
│   ├─ 用户希望模型自由发挥
│   │   └─ 导演四块结构 + 动作节拍自然描述
│   ├─ 时间分配没把握
│   │   └─ 导演四块结构 + 分镜节拍，不写秒数
│   └─ 用户要精确控制每一拍 / 有明确卡点
│       └─ 导演四块结构 + 五段式 + 时间线标记
│
└─ 多角色对话戏 / 复杂动作戏
    ├─ 同一机位、只需要错开动作和台词
    │   └─ 导演四块结构 + 分镜节拍，不写秒数
    └─ 严格正反打 / 硬切 / 遮挡 / 音效同步
        └─ 导演四块结构 + 时间线分段
```

判断的核心问题：**用户是不是想让模型自己掌握节奏**？

- 是 → 导演四块结构，动作自然描述
- 否，但时间分配没把握 → 分镜节拍
- 否，且卡点明确 → 时间线 + 五段式
