# 角色一致性维护清单（视频版）

本文件供 aigc-video-prompt skill 使用。视频比图片更容易丢一致性——同一段视频内、人物脸会逐渐变形；多镜头之间、角色细节会漂移。本文件给出怎么锁、用什么首帧、怎么写锚点。

## 视频里"丢一致性"的三种典型情况

| 情况 | 表现 | 主要原因 | 主要对策 |
| --- | --- | --- | --- |
| **单镜内变形** | 5 秒视频内、人脸从第 2 秒开始扭曲、发际线漂、瞳色变 | 模型在帧间预测时累计误差 | 加锚点句 + 控制时长 ≤5 秒 + 减少大幅动作 |
| **多镜间漂移** | 同一角色在镜头 A 和镜头 B 看起来不像同一人 | 每个镜头独立生成、没有跨镜参考 | 每个镜头的 prompt 都复述角色档案 + 同一参考图 |
| **变装/换景不一致** | 角色换衣服或换场景后、脸/身型也跟着变了 | 模型把"服装变化"误解为"角色变化" | 明确写"同一人物 + 换装/换景"、把不变的部分（脸、发型、配饰）列出来 |

## 首帧选用原则

视频 SKILL.md 的 Step 0 已经写了首帧质量检查。本节是"什么样的首帧最有利于保持一致性"的补充。

### 推荐首帧

| 类型 | 为什么好 |
| --- | --- |
| **正面或四分之三侧面、中近景** | 五官信息完整、模型有足够锚点 |
| **白底或简单背景的全身立绘** | 角色信息密度最高、背景不干扰 |
| **标准光线（无极端逆光、无大面积阴影）** | 不会因为光遮挡丢失五官信息 |
| **角色档案完整的图（瞳色/痣/配饰都清楚）** | 模型抓得到关键标志 |

### 不推荐首帧

| 类型 | 为什么差 |
| --- | --- |
| 背影 / 剪影 | 视频里"转过身"瞬间会重新定义角色 |
| 远景小人 | 五官信息不够、容易自由发挥 |
| 大特写只有局部（只有手、只有眼睛） | 没有全局参照 |
| 极端逆光 / 大阴影遮脸 | 五官隐藏、视频里"光线变化时"角色就变了 |
| 抽象/风格化（水彩、剪影艺术） | 模型在视频里很难维持非常规风格 |

### 不达标怎么办

直接用 SKILL.md Step 0 表里那三条路：放大 / 改图 / 重画。**不要在不达标首帧上跑视频**。

## 必锁的 12 个细节

与 aigc-image-prompt 的 character-consistency.md 同步。视频补充：

- **写法**：不要把 12 项全堆进 prompt（每段都堆会污染动作描述）。视频 prompt 里的"角色档案"段只写 5-6 项最关键的（发际线/瞳色/特殊标志/发型/服装/配饰），其余靠首帧图承载。
- **位置**：在【最终提示词】的"首帧是..."句子里、或者紧跟在首帧后的一段。

参考完整 12 项见 aigc-image-prompt 同名文件。这里列视频里"特别容易崩"的 5 项：

### 视频里崩得最快的 5 项

1. **发际线**：转头时最容易变形
2. **瞳色**：特殊瞳色（灰/绿/异色瞳）在视频中后段容易回归常态
3. **痣 / 标志**：动作幅度大时容易"消失"
4. **配饰位置**：耳坠在动作中容易"换边"或消失
5. **服装细节**（绣花、暗纹、特殊剪裁）：动态中会简化

针对这 5 项、在 prompt 里至少要明确写一遍。

## 锚点句模板（必加）

视频 prompt 末尾必须有锚点句。模板：

### 单镜锚点

```
整段保持人物面部特征、[标志性细节1]、[标志性细节2]、服装、发型、构图在整段视频中稳定不变。
```

填空示例：
```
整段保持人物面部特征、右眼下方小痣、左耳月牙耳坠、银白长直发、黑色对襟长袍稳定不变。
```

### 多镜锚点

如果是多镜头串联（比如同一角色的两条 5 秒视频后期拼接）：每条独立生成的 prompt 里都要复述完整锚点。

### 变装 / 换景锚点

```
角色为同一人物，仅 [服装/场景] 变化，面部特征、发型、[标志性细节] 保持稳定不变。
```

## 多镜头串联场景的特殊处理

用户做仿真人剧或精品广告时，常需要"同一角色出现在多个镜头"。注意事项：

### 1. 用同一张参考图

不要镜头 A 用首帧图 a、镜头 B 用首帧图 b。模型不会自动认这是同一个人。最稳的做法：

- 每个镜头都用同一张"标准照"（正视图或三视图里的正面）作为参考图 @图1
- 在 prompt 里写"角色 @图1"

### 2. 每个镜头独立写完整 prompt

不要写"接上一镜的场景"。每个镜头都从"首帧是..."开始，完整描述。

### 3. 注意"动作衔接的物理合理性"

镜头 A 末态 = 镜头 B 首帧状态。比如：
- 镜头 A 末态：女孩望向窗外
- 镜头 B 首帧：必须从"女孩望向窗外的同一姿态"开始

如果两个镜头之间状态对不上（A 是站着、B 是坐着、中间没交代），观众会觉得"跳了"。**这不是模型问题、是分镜设计问题**——副导演要在【镜头分析】里提醒用户。

### 4. 风格统一锚点

多镜头还要锁"风格"——色调、画风、光线基调。可以在每条 prompt 末尾加：

```
整段画面保持冷蓝青色调 / 胶片颗粒质感 / 低饱和度色调（沿用首帧风格）稳定一致。
```

## 反例集

### ❌ 视频 prompt 里只写"她"

错：她转头看窗外。

为什么错：模型不知道"她"是谁、参考首帧图但首帧细节会在动作中漂。

正：女孩（@图1）保持望货架的姿态约 1 秒、然后缓慢转头望向窗外。整段保持人物面部特征、银白长直发、左耳月牙耳坠稳定不变。

### ❌ 在没锁配饰的视频里期待配饰一致

错：（prompt 里没提耳坠）跑出来发现耳坠在视频中段消失了。

为什么错：模型没被告知"这是关键标志"、当成可有可无的装饰。

正：在 prompt 里至少提一次"左耳月牙耳坠"。

### ❌ 用模糊背影做首帧

错：用一张"女孩走在街上的背影"做首帧、然后让她"转过身"。

为什么错：转过身的瞬间、模型在"自由定义角色"。

正：先用正视图作为参考图、首帧选有正面或四分之三侧面信息的图。

### ❌ 多镜头但每个镜头用不同的参考图

错：镜头 A 参考图是"立绘"、镜头 B 参考图是"中近景"、镜头 C 参考图是"侧脸"。

为什么错：三张图模型可能识别为三个相似但不同的人。

正：选一张"标准照"用于所有镜头。其他参考图作为"动作参考""场景参考"分开用。

### ❌ 锚点句堆所有细节

错：保持人物面部特征、发际线、眉形、瞳色、瞳孔纹理、鼻形、唇形、痣、雀斑、发型、发色、刘海、耳坠、项链、戒指、服装、内搭、外套、腰带、裤子、鞋款、构图、光线、色调、背景所有元素稳定不变。

为什么错：堆 20+ 项、模型重点模糊。

正：挑 5-7 项最关键、其他默认靠首帧承载。例："保持人物面部特征、银白长直发、左耳月牙耳坠、黑色长袍、构图稳定不变"。
