# 视频问题诊断与修复（用户回投跑崩了时查）

本文档供 aigc-video-prompt skill 使用。当用户回投说"跑崩了"或"不稳"时，按问题类型查这里的修复策略。

适用范围：公开通用的视频问题诊断、原因判断和提示词修复策略。

---

## 五大问题决策树（先识别问题类型，再修）

| 问题类型 | 典型表现 | 主要成因 | 修复方向 |
| --- | --- | --- | --- |
| **画面摇晃** | 整段画面在颤抖、抖动、不稳 | 多个冲突的相机运动 / fast 这类未限定的关键词 | 简化为单一运镜 + 加节奏词 |
| **主体变形** | 肢体弯曲、多余手指、面部变形、物体融化 | 运动复杂度超过模型处理能力 | 降低运动复杂度 + 关键帧锁定 |
| **闪烁 / 时间伪影** | 纹理闪烁、亮度呼吸、边缘抖动 | 提示词冲突的光照指令 / 高频纹理 | 锚定单一光源 + 移除闪光元素 |
| **身份漂移** | 角色脸部变化、服装变化、Logo 变形 | 模型在长视频中"忘记"角色 | 一致性锚点 + 降低创意强度 |
| **风格不一致** | 风格中途漂移、画风混乱 | 多个竞争的风格关键词 | 应用 2+1 法则 |

---

## 问题 1：画面摇晃（Jitter / Shaking）

### 典型表现

- 整段画面在颤抖
- 主体边缘不停跳动
- 运镜路径混乱不平滑

### 根本原因（按出现频率排序）

1. **指定了多个冲突的相机运动**（最常见）
2. **使用了 fast 这类未限定的高风险关键词**
3. **同时要求快速相机移动 + 快速主体移动 + 复杂场景**

### 修复流程（按优先级）

**Step 1：简化为单一主相机指令**

检查 prompt 里有几个运镜动词。把超过 1 个的都删掉，只留最重要的那个。

- ❌ "推进、环绕、向上摇"
- ✅ "缓慢推进"

**Step 2：添加节奏控制词**

- 加 "缓慢"、"平滑"、"稳定"、"渐进"、"轻柔"
- 删除 "fast"、"快速"、"急速"（用具体动词替代——sprinting / vaulting）

**Step 3：追加防抖约束（如果模型支持英文 avoid 句式）**

- "avoid jitter and bent limbs"
- "no shake"

**Step 4：降低场景复杂度并缩短测试时长**

- 把 10 秒缩到 5 秒先跑通
- 减少画面里同时活动的元素数量

## 问题 2：主体变形 / 裁切（Deformation / Clipping）

### 典型表现

- 肢体弯曲、多余手指、缺手指
- 面部变形、五官扭曲
- 物体边缘融化

### 根本原因

运动复杂度超过模型处理能力。手部和面部细节渲染是 AI 视频最难的两块。

### 修复策略

1. **降低运动复杂度**：一次只做一个动作
2. **关键帧锁定**：在首帧或尾帧明确锁定关键姿势
3. **从中景而非特写构图**——特写暴露所有不对称，中景容错高
4. **取景规避**：让手部部分在画面外（部分显示手腕），或用浅景深虚化手部
5. **慢动作保护**：升格慢动作降低肢体跟踪出错概率
6. **流动元素掩盖**：头发、衣物、烟雾、雨水、花瓣飘落——既增美感又分散注意力（见 lighting-and-style.md "写实人像策略"）

### 不要做的事

- ❌ 不要写 "perfect hands"——模型不知道这是什么
- ❌ 不要在已经变形的主体上反复 rerun，结构性问题 rerun 救不回来——回去改 prompt

## 问题 3：闪烁 / 时间伪影（Flicker / Temporal Artifacts）

### 典型表现

- 纹理"爬行"——画面表面看起来在蠕动
- 亮度忽明忽暗（呼吸现象）
- 边缘闪烁

### 根本原因

时间一致性失败。最隐蔽也最顽固的问题。

### 修复策略（三层）

**Prompt 层（70% 案例靠这层解决）**：

1. **固定相机**——运镜越多越闪烁，先试固定
2. **锚定单一光源**——把"美丽的灯光"换成"左侧 45 度柔光从哑光墙面反射"
3. **移除高频纹理材质**——亮片、网格、密集花纹、金属反光在视频里特别容易闪
4. **避免冲突光照**——一段 prompt 里不要既写"warm sunlight"又写"cool moonlight"

**Reference 层（图片参考问题）**：

1. **参考图先放大再轻度去噪**——糊图是闪烁的常见源头
2. **统一参考图的曝光和色彩**

**Input 层**：

1. **降低 strength / 创意强度** 到 0.55 左右
2. **缩短镜头**到 4-6 秒——长视频闪烁概率显著增加

### ⚠️ 不要做的事

- ❌ **不要简单 rerun（无改动重新生成）**——结构性闪烁 rerun 救不回来。"累的时候我会习惯性地重跑，但这几乎没用"

## 问题 4：身份漂移（Identity Drift）

### 典型表现

- 角色面部在视频中逐渐变化
- 服装颜色 / 款式中途改变
- Logo 变形 / 配饰消失

### 根本原因

模型在长视频中"忘记"了角色——transformer 注意力机制的天然局限。第 4-5 个镜头开始出现"注意力疲劳"。

### 修复策略

1. **一致性锚点必加**（已经是 SKILL.md 默认）——"保持人物面部特征、服装、发型、构图在整段视频中稳定不变"
2. **降低创意强度 / 幻觉滑块** 到 0.3-0.4——强制 AI 紧贴参考图而非发明新细节
3. **关键帧注入**：每个新镜头的起始帧重新上传角色参考图
4. **缩短单镜时长**——6-8 秒能保持，超过 12 秒开始出现服装变换
5. **角色名词必须一致**——前后用同一个称呼，不要"the woman / she / the businesswoman"切换

### 多角色场景的特殊处理

如果 2 个角色在画面中靠太近，可能发生**融合故障**（Amalgamation Glitch）——AI 把两个人的面部特征混淆生成"混合人"。

- 用唯一标签（@CharacterA / @CharacterB）
- 不要描述角色相互接触或重叠
- 拥抱 / 打斗等近距离互动建议分别生成 + 后期合成

详见 `character-consistency.md`。

## 问题 5：风格不一致（Style Drift）

### 典型表现

- 风格中途漂移
- 画风混乱
- 多种美学元素打架

### 根本原因

多个竞争的风格关键词。

### 修复策略

1. **应用 2+1 法则**：最多 2-3 个风格引用，且必须在同一美学谱系（详见 `lighting-and-style.md`）
2. **锁定白平衡和光线方向**——光不稳整体就不稳
3. **移除多余形容词**——"电影感、史诗、惊艳"全删，用具体视觉描述替代
4. **风格词不要和内容词混在同一并列结构**——会污染（详见 aigc-image-prompt skill 主纪律）

## 生成前 60 秒理智检查清单

写完 prompt、点生成之前过一遍。这 60 秒能节省你之后几分钟的重跑等待。

### 必查 5 项

- [ ] **首帧明确写了**："首帧是 ___" 开头
- [ ] **只有一个主运镜指令**——超过 1 个直接抖
- [ ] **没有未限定的 "fast" / "快速"**
- [ ] **结尾收尾两句完整**："保持 ___ 稳定一致" + "不要出现字幕、不要出现 BGM"
- [ ] **参考图 @图N 标注正确**——元素首次出现紧跟 @图N

### 加分 5 项

- [ ] 拼写检查通过（一个错别字可能完全跑偏）
- [ ] 参考图引用顺序对（@图1 是不是写错成 @图2）
- [ ] 画幅 / 时长符合目标平台（例如短视频竖版 9:16）
- [ ] 单镜时长 ≤ 模型上限（查 model-capabilities.md）
- [ ] 没在重复参考图已有的视觉信息（墙、灯、布局、服装颜色）

### 完整的 11 项预生成检查（高要求场景）

1. 提示词结构完整（首帧 + 运镜 + 动作链 + 锚点 + 收尾）
2. 避免了复合相机运动（运镜法则 1）
3. 节奏词替代技术参数（运镜法则 2）
4. 相机和主体动作分开（运镜法则 3）
5. 风格描述词数量 ≤ 3 且同谱系（2+1 法则）
6. 一致性锚点已加
7. 参考图格式正确（JPG/PNG/MP4/MP3）
8. 参考图数量 ≤ 12（9 张图 + 3 段视频）
9. 参考视频总时长 ≤ 15 秒
10. 画幅符合目标平台
11. 时长设置合理（首次测试 5-8 秒，稳定后再延长）

## 迭代优化的"一次一变"原则

跑出来不满意 → **每次只改一个变量**。

- ❌ 同时改运镜 + 时长 + 光线 + 风格 → 你不知道是哪个改变起作用
- ✅ 锁定其他不动，只改光线 → 跑一次对比，确定光线方向是否改善

调整变量的优先级排序：

1. 相机角度 / 运动
2. 运动强度
3. 风格 / 光照
4. 约束条件
5. 时长

如果 4-5 次"一次一变"还跑不出满意结果，回到 Step 0（首帧图质量检查，见 SKILL.md），可能是图本身的问题。

## 务实面对的极限（不要硬攻）

不是每个问题都能正面解决。AI 视频当前的硬边界：

| 难做的事 | 替代方案 |
| --- | --- |
| 手部特写 + 复杂手指动作 | 取景虚化手部、改产品反应特写 |
| 写实人像 + 双人亲密对话 | 加流动元素、慢动作、侧面跟拍 |
| 多角色近距离互动（拥抱/打斗） | 分别生成 + 后期合成 |
| 镜头切换 / 闪回 / 剪辑 | 拆段后期接 |
| 复杂多乐器音乐卡点 | 用简单节拍音乐参考 |
| 多语言对白口型同步 | 英语 UGC 可用，中文建议剪映兜底 |
| 单镜超长（≥ 15 秒）保持一致性 | 拆段 + 关键帧注入 |

"有时候一个好的镜头设计比一百个负面提示词更有效。" —— 与其硬攻手部变形，不如用景别和构图规避。
