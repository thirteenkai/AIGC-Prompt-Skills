# aigc-video-prompt 评估集

本文件供 aigc-video-prompt skill 使用。改 skill 后必跑回归。

---

## 怎么用

每次大改 SKILL.md 或 references 后、新开一个对话、用例的"输入"作为用户输入、对照"通过标准"打勾。全过才能合并修改。

---

## 评估样例（9 个）

### 用例 1：图像资产质量检查、首帧不达标

**输入**（带一张明显有 AI 瑕疵的图，比如手指数错或脸扭曲）：
> 让她转头看窗外

**通过标准**：
- [ ] 副导演**没有直接出 prompt**
- [ ] 触发 Step 0 首帧质量检查
- [ ] 明确指出瑕疵在哪（脸 / 手 / 边缘等）
- [ ] 根据瑕疵类型给出对应路径（ComfyUI 放大 / 回 aigc-image-prompt 改图 / 重画）
- [ ] 没在烂图上凑合写 prompt

### 用例 2：单镜动作生成、首帧达标但信息不足

**输入**（带一张干净的女孩在便利店分镜图）：
> 她要转头

**通过标准**：
- [ ] 首帧检查通过
- [ ] 主动追问 3-5 个问题（不能全部都问）
- [ ] 追问含：动作慢急 / 时长 / 运镜 / 转头后干什么 / 情绪
- [ ] 用人话问、不出现"推拉摇移"等术语
- [ ] 用户回答后才出最终 prompt

### 用例 3：单镜动作生成、含多动作链

**输入**（带一张干净首帧）：
> 她玩手机、突然吸气、憋了一下、打喷嚏、然后愣住、约 12 秒

**通过标准**：
- [ ] 副导演**不擅自拆段**（用户给的整体方案）
- [ ] 出单镜 prompt（≥10 秒）
- [ ] 推荐 Seedance（多动作链强项）
- [ ] 【最终提示词】使用导演四块结构：素材锚定 / 镜头与动作 / 场景光影 / 声音与约束
- [ ] 【最终提示词】放在独立 `text` 代码块里，代码块内只有可复制提示词本体
- [ ] 【镜头与动作】优先按动作节拍组织；只有时间锚点明确时才写秒级时间线，不把每个短动作都机械写成"约 X 秒"
- [ ] 【声音与约束】含环境音、关键动作音效、锚点稳定句、"不要出现字幕、不要出现 BGM"
- [ ] 在【判断和取舍】里**简短**提一句"如果 X 段不稳可以拆"做兜底，不主动出拆段方案

### 用例 4：单镜动作生成、含情绪词

**输入**：
> 她接到电话、知道有人去世、很失落、5 秒

**通过标准**：
- [ ] 触发情绪追问（"失落"必须下沉到可见动作）
- [ ] 追问"通过什么动作 / 生理变化 / 姿态变化感受"
- [ ] 【最终提示词】里不出现"失落"标签、只有动作和姿态
- [ ] 【最终提示词】放在独立 `text` 代码块里，方便直接复制
- [ ] 使用导演四块结构，但 5 秒单一情绪镜头不强行写满秒级时间线
- [ ] 【声音与约束】含环境音 / 关键动作音效；没有明显动作音效时明确写"无明显动作音效"

### 用例 5：首尾帧过渡

**输入**（带两张图：A 望窗外、B 推门背影）：
> 从望窗外到推门走出去、5 秒

**通过标准**：
- [ ] 副导演追问过渡类型（一气呵成 / 有停顿）、运镜、强调瞬间
- [ ] 用户回答后【镜头分析】含"过渡逻辑"段
- [ ] 【最终提示词】首句"首帧是..."
- [ ] 【最终提示词】放在独立 `text` 代码块里，不混入工具建议
- [ ] 使用导演四块结构，并在【素材锚定】里说明 @图1 / @图2 职责
- [ ] 推荐工具说明（首尾帧空间过渡可偏向可灵）
- [ ] 锚点稳定句 + 不要字幕 / 不要 BGM 末句

### 用例 6：触发参考图标注规则

**输入**（带 2 张参考图）：
> @图1 是房间、@图2 是角色、让角色在房间里坐着喝咖啡、5 秒

**通过标准**：
- [ ] 【素材锚定】说明 @图1 锁定空间、@图2 锁定角色
- [ ] 【最终提示词】里元素第一次出现紧跟 @图N
- [ ] 不重复描述参考图已有的视觉信息（墙面、灯光、布局、服装）
- [ ] 只写动作 / 运镜 / 新元素（咖啡杯算新元素）
- [ ] 即使 prompt 简短，也保留导演四块结构

### 用例 7：简单 5 秒首帧动作

**输入**（带一张干净首帧）：
> 让他慢慢抬头看镜头，5 秒

**通过标准**：
- [ ] 推荐 Seedance（人物表演 / 眼神 / 微表情）
- [ ] 使用导演四块结构
- [ ] 【最终提示词】放在独立 `text` 代码块里
- [ ] 【镜头与动作】自然描述动作，不强行拆成 0-1 / 1-2 / 2-3 秒
- [ ] 【场景光影】只写主光源和稳定性，不堆空泛风格词
- [ ] 【声音与约束】含环境音、呼吸 / 衣料等轻微音效、不要字幕、不要 BGM

### 用例 8：12-15 秒 Seedance 多动作链

**输入**（带一张干净首帧）：
> 角色在走廊里边走边接电话，听到坏消息后停下，回头看门口，最后把手机慢慢放下，15 秒

**通过标准**：
- [ ] 推荐 Seedance（15 秒多动作链 + 表演 + 道具操作）
- [ ] 不预防性拆段
- [ ] 【最终提示词】放在独立 `text` 代码块里，代码块外再写工具与参数建议
- [ ] 【镜头与动作】优先使用动作节拍；只有时间锚点明确时才用秒级时间线
- [ ] 明确机位、人物比例、手机道具状态和尾帧动作
- [ ] 【声音与约束】含走廊环境音、脚步、手机摩擦、停步音效；不生成清晰电话内容，除非用户要求
- [ ] 没有出现参考资料名、私人案例、历史素材 ID 或本地路径

### 用例 9：15 秒多动作对白戏，不预防性 A/B

**输入**（有多个资产或纯文生均可）：
> 小动物摊主在街边小摊制作甜点，急躁顾客先点单，中途催促两次，摊主继续加入馅料、裹配料，被催烦后短暂表达不满，最后盖一个标志印记、装袋递给顾客，顾客吐槽收尾。15 秒，16:9。

**通过标准**：
- [ ] 判断为 15 秒内 Seedance 可承载的单段多动作对白戏，不拆 A/B
- [ ] 如果时序没有明确卡点，【镜头与动作】使用"分镜一 / 分镜二 / 分镜三"或"起手 / 制作 / 反应 / 收尾"，不硬写 0-3 / 3-5 秒
- [ ] 如果输出秒级时间线，必须说明依据：用户给了时间、硬切/遮挡/音效同步点或尾帧卡点
- [ ] 台词错开写，不让多个角色同时说话
- [ ] 代码块里只有一段【最终提示词】，不出现【视频 A】/【视频 B】
- [ ] 【声音与约束】含环境音、制作动作音效、角色反应音效，不写 BGM
