# 输入信息与剧情检查

## 轻量版输入模板

```text
【故事板需求】

1. 这段剧情讲什么？
2. 观众必须看懂什么？
3. 必须出现哪些画面或动作？
4. 角色的情绪变化是什么？
5. 台词 / 字幕 / 音效有哪些？
6. 有什么绝对不能错？
```

一句话版：

```text
请提供：剧情完整描述 + 观众必须看懂的点 + 必须出现的动作画面 + 角色情绪变化 + 台词/字幕/音效 + 绝对不能错的限制。
```

## 完整信息卡

```text
【剧情信息卡】

1. 账号/栏目：
2. 账号风格：
3. 本段剧情：
4. 目标时长：
5. 目标画幅 / 成片比例：
6. 主要角色：
7. 场景与关键道具：
8. 本段情绪变化：
9. 必须出现的动作或画面：
10. 不能出现的内容：
11. 特殊镜头或表现方式：
12. 台词、字幕、音效说明：
13. 参考资产映射：
14. 是否允许拆成上下两段视频：
```

## 必须检查

- 观众必须看懂什么
- 角色是否完整
- 动作因果是否清楚
- 不可缺少画面是否明确
- 台词、后期字幕、音效是否区分
- 是否有容易误解的点
- 时长是否对应合理镜头数
- 目标时长是否承载得下完整叙事，而不是只看镜头数量
- 如果承载不下，是否需要拆成 A/B 两段视频
- 目标画幅是否明确；若平台或账号要求 9:16、16:9、1:1 等，提示词必须跟随该比例，不得写死

## 时长承载判断

不要只用“10 秒 3-4 镜头、15 秒 5-7 镜头”判断能不能做。先数叙事节拍：

- 一个叙事节拍 = 一个观众必须看懂的事件、状态变化、情绪转折、台词回应、道具状态变化或因果触发。
- 10 秒通常只能承载 3-4 个清晰节拍。
- 15 秒通常只能承载 5-7 个清晰节拍。
- 如果剧情需要观众看懂“先发生 A，所以角色才做 B，最后 C 得到解决”，A/B/C 都必须在镜头里成立。

判断输出格式：

```text
【时长承载判断】
结论：15 秒内可完成 / 建议拆成 A/B 两段 / 信息不足
依据：
1. 关键节拍数量：
2. 必须保留的因果：
3. 最容易被压没的画面：
置信度：高 / 中 / 低
```

拆分标准：

- 节拍数超过 7 个，默认建议拆分。
- 有 2 轮以上对话，并且每轮对话都触发角色动作变化，优先拆分。
- 有“发现问题 -> 追赶/拦截 -> 等待结果 -> 问题解决”链条，通常拆分。
- 只有“递出 -> 听到信息 -> 收回/解释 -> 再递出”这类短链条，通常 10-15 秒可完成。

## 不合格输出

```text
【剧情信息不足，暂不生成提示词】
缺口：
为什么会影响故事板：
请补充：
```
