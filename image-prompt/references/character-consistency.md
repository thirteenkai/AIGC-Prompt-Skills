# 角色一致性维护清单（图片版）

本文件供 image-prompt skill 使用。AI 出图最容易在"角色"维度崩——同一个角色每次出图细节都漂。本文件给出哪些细节必须锁、怎么锁、双步工作流的标准。

## 双步工作流（必须遵守）

原则：**新角色先给正视图提示词锁角色，再转三视图，不要一步到位跑三视图**。

| 步骤 | 做什么 | 产出 |
| --- | --- | --- |
| **第 1 步：正视图锁角色** | 用 image-prompt 场景 A 生成一张正面立绘（标准光线、中性表情、白底） | 角色的"原始档案"，所有后续生成的视觉锚 |
| **第 2 步：转三视图** | 用 image-prompt 场景 C，@图1 = 正视图，文字 prompt 只讲"生成同一角色的左侧 45 度、正侧、背面三视图"+ 极简锚点 | 三视图，用于后续动作/场景图的参考库 |
| 第 3 步及之后 | 场景图、动作图、视频首帧——都以三视图 + 正视图为参考库 | 一致性的角色资产库 |

**纪律**：跳过第 1 步直接跑三视图，相当于让模型同时定义角色 + 演绎角色，结果就是三视图里的三个人长得不一样。

## 必锁的 12 个角色细节（按崩坏频率排序）

按"AI 出图最容易把这个画错"的顺序。**前 5 个必锁**，后面的视角色重要度选锁。

### 1. 发际线形状（崩坏率★★★★★）

为什么崩：模型对发际线的"曲线 + 位置"理解最弱、每次出会重新猜。
怎么锁：写明"发际线是平的、还是 M 字形、美人尖、圆弧、不规则"。例："美人尖发际线、额头中央有明显的尖角"。

### 2. 瞳孔颜色和形状（崩坏率★★★★★）

为什么崩：模型默认偏棕黑、特殊瞳色（灰、绿、琥珀、异色瞳）每次跑都会回归常态。
怎么锁：精确颜色 + 是否有特殊纹理 + 异色瞳要分别标注。例："冷灰蓝瞳孔、右眼有一道纵向琥珀色纹理"。

### 3. 痣 / 雀斑 / 疤的位置（崩坏率★★★★★）

为什么崩：装饰性细节、模型容易漏或挪位置。
怎么锁：用空间锚点说位置 + 形状大小。例："右眼下方一颗小痣、距眼角约 1cm"、"鼻梁上有淡淡雀斑"、"左侧眉毛尾部有一道浅疤"。

### 4. 发色和发型（崩坏率★★★★）

为什么崩：发色容易"漂"成相近色（米白漂成金、银漂成灰）；发型的具体造型（齐刘海/八字刘海、单边耳后、马尾位置）容易简化。
怎么锁：发色用具体色名（不用"浅色"）、发型用具体造型词。例："银白长直发及腰、齐刘海、左耳露出"。

### 5. 眉形（崩坏率★★★★）

为什么崩：眉毛是脸部"性格"最主要的载体、漂一点整个气质就变。
怎么锁：粗细 + 弧度 + 起止位置。例："细弯眉、眉峰偏外侧、眉尾微微下垂"。

### 6. 鼻形（崩坏率★★★）

为什么崩：侧面尤其容易崩。
怎么锁：高低 + 鼻头形状。例："鼻梁挺直但不夸张、鼻头圆润不尖"。

### 7. 嘴唇厚度 / 唇形（崩坏率★★★）

怎么锁：薄/厚 + 唇峰形状。例："薄唇、唇峰明显呈 M 形"或"嘴唇饱满、下唇略厚"。

### 8. 体型 / 身材比例（崩坏率★★★）

怎么锁：身高印象 + 体型词。例："偏瘦高、肩膀窄、肢体修长"。

### 9. 服装具体款式（崩坏率★★★★）

为什么崩：尤其多层穿搭、非常规款式容易简化。
怎么锁：每一件衣物单独描述 + 材质 + 关键细节。例："黑色对襟长袍、立领、袖口绣银色暗纹、腰间束白色丝带"。

### 10. 配饰位置和形态（崩坏率★★★★）

为什么崩：配饰是"装饰性"信息、模型容易省略或挪位置。
怎么锁：每一个配饰都点名 + 戴在哪 + 什么材质。例："左耳戴一只银色月牙耳坠、右耳无配饰、颈间戴一条带玉石坠子的细银链"。

### 11. 鞋款（崩坏率★★★）

为什么崩：下半身容易被裁掉或简化。
怎么锁：鞋型 + 颜色 + 鞋帮高度。例："黑色及踝皮靴、低跟"。

### 12. 整体气质形容词（崩坏率★★）

不是必锁项，但用 1-2 个气质词能帮模型校准。例："清冷"、"温润"、"凌厉"。注意：不要堆砌，最多 2 个，否则模型会迷茫。

## 锁定话术模板

放在【最终提示词】里的"角色档案"段：

```
[角色名]，[年龄段]，[性别]，[整体气质 1-2 个词]。
五官特征：[发际线]、[瞳色]、[眉形]、[鼻形]、[唇形]，[特殊标记如痣/雀斑/疤的位置]。
发型：[长度]、[颜色具体色名]、[造型]。
体型：[高/矮/瘦/壮]、[肩宽]、[整体比例]。
服装：[每一件单独写：颜色 + 款式 + 材质 + 关键细节]。
配饰：[每一个单独写：戴在哪 + 材质 + 形态]。
鞋：[款式 + 颜色 + 鞋帮]。
```

**多角色场景**：每个角色单独一段、用名字分隔。不要混着写，否则模型会把 A 角色的特征串到 B 上。

## 配饰 / 道具的特殊处理

### 重要配饰的命名

如果某个配饰是"角色识别标志"（比如月牙耳坠、玉佩、特殊戒指、独特纹身），给它一个**短名字**并在 prompt 里重复使用。例："标志性的月牙耳坠"——比每次都描述形态更稳。

### 多图保持配饰一致

在多图工作流（角色出现在不同场景）里，每张图都要把这些标志性配饰写进 prompt。模型不会自动"记住"上一张图，每张是独立生成。

### 不要让配饰"挪位"

错的写法：戴着月牙耳坠（没说戴在哪边、模型每次猜）
对的写法：左耳戴一只银色月牙耳坠、右耳无配饰

## 反例集

### ❌ 用"同一个人"或"上一张图里的人"

错：和上一张图里的女孩是同一个人。

为什么错：模型每张图独立生成、看不到上一张。

正：把角色档案完整粘贴进 prompt。

### ❌ 笼统的发色

错：浅色头发。

为什么错："浅色"在模型里可能是浅金、米白、灰白、亚麻、浅棕——每次跑都换。

正：银白色长直发、不带蓝调也不带黄调。

### ❌ 漏掉痣 / 标志

错：女孩的脸部档案（写了发型瞳色眉形）。

为什么错：写了大部分但漏了"右眼下方那颗痣"。下次生成就没了。

正：每次提到该角色、所有标志性细节都要在 prompt 里复述一遍。

### ❌ 一次跑三视图

错：直接 prompt"白发少年的正面、侧面、背面三视图"。

为什么错：模型同时定义+演绎、三个视图的角色长得不一样。

正：先跑正视图、确认锁定 → 用正视图作 @图1 + 极简 prompt 生成三视图。

## 与 video-prompt 的衔接

视频比图片更容易在"长时间运动中"丢失角色一致性。给视频 skill 的首帧用图时：
- 优先用角色档案完整的"正视图"或"接近正视图的中近景"
- 如果首帧本身角色细节不完整（背影、剪影、远景），视频里角色细节会自由发挥
- 视频 prompt 里要再次复述关键标志（"标志性月牙耳坠""右眼下小痣"）+ 加锚点句"保持人物面部特征、服装、配饰、发型在整段视频中稳定不变"
